# AI Usage - SplitFair

This document outlines the AI tools, prompts, mistakes identified, and corrections made during the development of SplitFair.

---

## 1. AI Tools & Environments
* **AI Coding Companion**: Gemini (via Antigravity).
* **Workspace Integration**: Direct codebase read/write, terminal execution capabilities, and test-running mechanisms.

---

## 2. Prompts Used
* **Initial Planning Prompt**: Analyzing project specs and generating a relational database schema.
* **Component Code Generation Prompts**: Requesting models, views, and frontend pages matching the approved implementation plan.
* **Verification Prompts**: Running test suites and validating Vite compiles.

---

## 3. AI Mistakes & Corrections

The following three issues were generated by the AI during implementation and corrected to ensure a production-ready deployment.

### Example 1: Missing Import of `uuid` in API views (`views.py`)
* **Mistake**: Inside `backend/expenses/views.py`, the AI generated the `CSVUploadView` class which names files uniquely using `uuid.uuid4()`. However, the AI forgot to include the `import uuid` statement in the imports header.
* **Impact**: If left uncorrected, any attempt to upload a CSV spreadsheet would crash the server with a `NameError: name 'uuid' is not defined`.
* **Correction**: During verification, the system ran `python manage.py check`. The error was flagged, and the developer added `import uuid` on line 2 of `backend/expenses/views.py`.

### Example 2: Missing Group Creator Membership
* **Mistake**: In the initial model and view drafts, the AI implemented `GroupViewSet.perform_create` to create a new group. However, it forgot to add a corresponding `GroupMembership` row for the creator.
* **Impact**: The creator of a group could see the group, but could not be added as a payer or participant because they were technically not recorded in the membership table, causing key balance calculations to return null values.
* **Correction**: Modified `GroupViewSet.perform_create` in `views.py` to atomically create a `GroupMembership` record for `self.request.user` on group creation.

### Example 3: Invalid Tool Call with `ArtifactMetadata` for Project Files
* **Mistake**: The AI called the `write_to_file` tool to create `tailwind.config.js` in the frontend codebase directory while passing the `ArtifactMetadata` parameter (which defines user-facing summaries for the chat UI).
* **Impact**: The sandbox execution system rejected the tool call with `invalid_args`, noting that `ArtifactMetadata` is strictly reserved for documents inside the central `/brain/` artifact folder.
* **Correction**: The AI corrected its parameters, removed the `ArtifactMetadata` block from its command call arguments, and successfully wrote the Tailwind configuration.
